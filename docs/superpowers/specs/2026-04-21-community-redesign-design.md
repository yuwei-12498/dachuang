# 社区精品化重构设计文档

> 面向 Web 端社区改版；当前版本已完成方向确认，可直接作为实施与验收基线。

## 1. 背景

当前社区能力已经具备基础的公开分享、点赞、评论、详情查看，但整体仍停留在“公开路线列表 + 轻互动 MVP”阶段，主要问题集中在：

- 社区首页更像功能列表，缺少可逛性、节奏感与内容氛围。
- 帖子卡片字段密度过高，信息主次不清，页面视觉杂乱。
- 社区治理能力不完整，管理员无法管理全部帖子，作者无法删除自己的帖子，也不能置顶自己帖下高价值评论。
- 当前“公开到社区”更像状态切换，不像真正的社区发布动作。

本次重构目标是将社区升级为 **围绕路线卡片展开的精品旅行分享社区**：既保留路线数据价值，又建立内容流体验、轻治理能力与后续扩展空间。

## 2. 目标

### 2.1 业务目标

1. 让社区首页从“可用”升级到“愿意逛”。
2. 让用户可以围绕路线内容进行轻社交，而不是围绕参数做阅读负担。
3. 补齐最小可行治理闭环：
   - 管理员可删除任意帖子、设置/取消全站置顶
   - 作者可删除自己的帖子、置顶自己帖子下的 1 条一级评论
4. 将“公开到社区”升级为“发布路线帖”，提升发布动作的社区感与完成感。

### 2.2 体验目标

- 风格关键词：**高级、清爽、有呼吸感、内容导向、轻运营感**
- 首页采用 **双列图文平衡型卡片流**
- 社区定位为 **社交动态流**，但不走碎片化杂乱风格
- 帖子内容固定为 **路线分享帖**，不扩展为纯图文泛帖子

## 3. 范围

### 3.1 本次范围（P0）

- Web 社区首页重构
- Web 社区详情页重构
- 发布路线帖流程升级
- 管理员全站删帖 / 全站置顶
- 作者删自己帖子 / 置顶 1 条一级评论
- 新增后台“社区管理”页面

### 3.2 下一阶段范围（P1）

- 我的发布
- 热门榜 / 专题页 / 推荐机制增强
- 从帖子详情一键收藏到我的行程
- 基于帖子路线生成我的版本

### 3.3 明确不做

- 审核流
- 举报系统
- 关注/粉丝
- 私信
- 作者主页
- 独立纯图文发帖
- 评论治理后台

## 4. 用户角色与权限

| 角色 | 能力 |
| --- | --- |
| 游客 | 查看首页、查看详情 |
| 登录用户 | 点赞、评论、发布路线帖 |
| 发帖作者 | 删除自己的帖子；置顶自己帖子下的 1 条一级评论 |
| 管理员 | 删除任意帖子；设置/取消全站置顶 |

权限原则：

1. 作者只管理自己的内容。
2. 管理员只做轻治理，不做审核流。
3. 治理入口尽量收敛到详情页菜单与后台管理页，不污染普通用户浏览界面。

## 5. 内容模型

社区内容固定为 **路线分享帖**，由已有路线快照生成，不支持独立纯图文发帖。

每篇帖子核心由两部分组成：

1. **路线卡片信息**
   - 标题
   - 封面图
   - 分享语
   - 主题标签
   - 时长 / 预算 / 节点数
   - 时间线内容
2. **社区状态信息**
   - 作者
   - 点赞数
   - 评论数
   - 是否全站置顶
   - 作者置顶评论
   - 发布时间 / 更新时间

## 6. 状态设计

建议在帖子层补齐以下状态：

- `isPublic`：是否公开
- `isDeleted`：是否已删除
- `isGlobalPinned`：是否全站置顶
- `globalPinnedAt`：全站置顶时间
- `globalPinnedBy`：执行管理员
- `pinnedCommentId`：作者置顶评论 ID

规则：

- 首页置顶专区展示 `isGlobalPinned = true` 的帖子
- 进入置顶专区的帖子不在普通动态流重复出现
- 每篇帖子最多只能有 1 条作者置顶评论
- 作者置顶评论只允许一级评论，不允许楼中楼回复

## 7. 首页信息架构

首页采用四层结构：

### 7.1 Hero 区

内容：社区标题、一句副标题、主 CTA（发布路线帖）、次 CTA（查看热门路线）。

### 7.2 全站置顶专区

- 独立展示 1~3 条置顶帖
- 大卡样式
- 明确标记“置顶推荐”
- 不与普通流混排

### 7.3 轻量筛选条

- 排序：最新 / 热门
- 标签：夜游 / 美食 / 亲子 / Citywalk / 拍照 等
- 搜索：标题 / 分享语
- 视觉采用吸附式胶囊标签，避免后台筛选器感

### 7.4 双列动态流

双列、图文平衡型卡片为主体，统一骨架，允许轻微高度差，但不做完全失控的瀑布流。

## 8. 首页卡片规范

卡片固定字段顺序：

1. 封面图
2. 标题（最多 2 行）
3. 分享语摘要（最多 2 行）
4. 标签区（主题标签最多 3 个）
5. 参数区（时长 / 预算 / 节点数）
6. 作者
7. 点赞 / 评论

不在首页卡片展示：完整时间线、过多节点细节、复杂管理状态、过多操作按钮。

## 9. 详情页结构

详情页采用“主内容列 + 侧边辅助列”的阅读流结构：

1. **Hero 头图区**：大封面、标题、作者、时间、点赞评论、更多操作。
2. **路线摘要区**：分享语全文、时长 / 预算 / 节点数、主题标签、路线摘要。
3. **路线时间线区**：每一站单独卡片展示时间、点位、停留、花费、推荐理由。
4. **评论区**：作者置顶评论在前，普通评论在后。
5. **底部行动区**：点赞、返回社区，预留收藏入口。

## 10. 发布路线帖流程

当前“公开到社区”升级为“发布路线帖”，流程改为 3 步：

1. **确认路线版本**：当前选中方案确认 + 封面与核心参数预览
2. **补充内容**：标题、分享语 / 推荐理由、标签
3. **发布预览**：预览首页卡片与详情头图后确认发布

发布成功后给出两个动作：**查看帖子**、**去社区首页**。

## 11. 治理设计

### 11.1 作者能力

- 删除自己的帖子
- 置顶自己帖子下的 1 条一级评论
- 替换已置顶评论时二次确认

### 11.2 管理员能力

- 删除任意帖子
- 设置全站置顶
- 取消全站置顶

### 11.3 后台结构

新增“社区管理”菜单，页面包含：

- 概览指标：总公开帖子数、置顶帖数、今日新增（可选）、今日互动（可选）
- 筛选：标题 / 作者 / 状态
- 列表字段：封面、标题、作者、发布时间、点赞、评论、状态
- 操作：设为置顶 / 取消置顶 / 删除

## 12. API / 数据层建议

新增或调整：

- 删除帖子
- 设置全站置顶
- 取消全站置顶
- 置顶评论
- 取消置顶评论
- 社区列表支持 `latest` / `hot`
- 社区列表支持排除已置顶帖子
- 管理后台分页查询帖子

## 13. 验收标准

### 13.1 首页

- 首页存在 Hero 区、置顶专区、筛选条、双列动态流
- 置顶帖不与普通流重复出现
- 卡片字段不超密，标题/分享语行数受控

### 13.2 详情页

- 页面分区明确
- 置顶评论固定在评论区顶部
- 管理入口按角色显示

### 13.3 权限

- 作者仅能删除自己的帖子
- 作者仅能置顶自己帖子中的 1 条一级评论
- 管理员可删除任意帖子
- 管理员可设置/取消全站置顶

### 13.4 发布

- 用户从结果页/历史页进入发布流程
- “公开到社区”升级为“发布路线帖”
- 发布完成后可直接查看帖子
